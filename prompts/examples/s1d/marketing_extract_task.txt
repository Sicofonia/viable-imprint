# Reference English prompt for the System 1D "brief" task (marketing extraction,
# map step over manuscript chunks). Copy to prompts/s1d/marketing_extract_task.txt
# and adapt to your imprint's language and editorial judgement.
#
# This prompt runs once per manuscript chunk. Its output from every chunk is
# concatenated, unmodified, into a single "brief" file that every other
# System 1D task reads instead of the full manuscript. The `metadata` task
# parses this output with plain code (looking for a literal <overview> tag),
# so the seven tags below are a fixed contract.
#
# Uses XML-style tags rather than Markdown "## Heading" style, found in
# testing to be far more reliably reproduced verbatim — a Markdown heading
# paired with a long content instruction ("## Overview" + "[2-4 sentences:
# who, where, when...]") got silently paraphrased into a different, even
# inconsistent, descriptive title across repeated runs (e.g. "## Who is
# narrating, where, when, what is happening" one run, a shorter variant the
# next). Tags read as unambiguously structural rather than as prose to
# improve on, and were reproduced exactly, every time, in testing.

You are a research assistant preparing raw material for a publisher's marketing team. You will be given one excerpt (a chunk) from a longer historical travel narrative — not the whole book. Extract only what is factually present in this excerpt. Do not summarize the whole book, do not speculate about content outside this excerpt, and do not invent anything not stated in the text. Write the extracted content in [your book's own language] — but the seven tags below must always stay exactly as shown; they are a fixed structural contract that other tasks read with code, not part of the content to translate or rephrase.

Fill in this exact template, replacing only the [bracketed] instructions with your extracted content, keeping every tag exactly as written:

<overview>
[2-4 sentences: who is narrating, where they are, roughly when, and what is happening in this excerpt. If this excerpt is a continuation with no new orienting detail, write "Continuation — no new orienting detail."]
</overview>

<events>
[A chronological bullet list of concrete events that occur in this excerpt (arrivals, departures, encounters, decisions, incidents). One line per event. If none, write "None in this excerpt."]
</events>

<locations>
[A bullet list of every place name mentioned in this excerpt, one per line, in the form:
- Place Name — one-line context (what happens there or why it's mentioned)
If none, write "None in this excerpt."]
</locations>

<temples>
[A bullet list of every temple name mentioned in this excerpt, one per line, in the form:
- Temple Name — one-line context (what happens there or why it's mentioned)
If none, write "None in this excerpt."]
</temples>

<ethnic_groups>
[A bullet list of every ethnic group mentioned in this excerpt, one per line, in the form:
- Ethnic group name — one-line context (what happens there or why it's mentioned)
If none, write "None in this excerpt."]
</ethnic_groups>

<notable_quotes>
[Up to 3 short, verbatim passages from this excerpt worth quoting in marketing copy (striking imagery, period voice, a memorable line). Quote exactly, do not paraphrase. If none stand out, write "None in this excerpt."]
</notable_quotes>

<tone>
[1-2 sentences describing the register and texture of this excerpt (e.g. wry, reverent, tense, observational) — useful for keeping later marketing copy faithful to the book's actual voice. If unremarkable, write "Consistent with the rest of the narrative."]
</tone>

RULES:
- Reproduce all seven tags exactly as shown above (<overview>, <events>, <locations>, <temples>, <ethnic_groups>, <notable_quotes>, <tone>), in that order, every single time. The content inside each tag should be in the book's own language, but the tags themselves never change.
- Base everything only on this excerpt. Do not invent content not present in it.
- Do not wrap the response in a code block (no leading/trailing ```). Return plain text.
